Cắm MCP đầu tiên của bạn vào Claude

Trong ~15 phút, bạn sẽ kết nối Filesystem MCP server (chính chủ Anthropic) vào Claude. Sau đó Claude có thể đọc, tìm và ghi file trong một thư mục bạn chỉ định — qua chuẩn MCP. Không cần viết code.

⏱ ~15 phút 🟢 Cho người mới — không cần kinh nghiệm ⌨️ Không cần code 🖥️ Windows / macOS

Ở phần Case study (§3) bạn đã thấy build và vận hành một MCP ra production khó thế nào. Lab này đi hướng ngược lại — phần dễ và phổ biến nhất: dùng một MCP có sẵn. Đây cũng chính là lifecycle initialize → list → call (slide "Lifecycle") chạy thật trước mắt bạn.

1Mục tiêu và kết quả đạt được

Kết thúc lab, bạn sẽ tự tay làm được điều này — hỏi Claude bằng tiếng Việt và nó thao tác file thật:

Bạn:   Liệt kê các file trong thư mục mcp-lab của tôi.
Claude: (gọi tool list_directory)  →  note1.md, note2.md

Bạn:   Đọc và tóm tắt tất cả file trong thư mục đó.
Claude: (gọi tool read_multiple_files)  →  "Hai ghi chú nói về..."

Bạn:   Tạo file summary.md tổng hợp các ghi chú đó.
Claude: (gọi tool write_file)  →  đã tạo summary.md ✅
✅ Thành quả: một Claude có thể "nhìn thấy" và làm việc với thư mục của bạn — đó là MCP đang hoạt động. Bạn cũng sẽ hiểu vì sao server chỉ được phép đụng đúng thư mục bạn cho phép (bài học "tool-surface = ranh giới bảo mật" của §4, ở dạng cụ thể nhất).

2Chuẩn bị — đúng 3 thứ

a

Node.js ≥ 18 (để chạy npx)

Filesystem server chạy bằng npx — công cụ đi kèm Node.js. Mở terminal (Windows: PowerShell; macOS: Terminal) và kiểm tra:

node --version
✅ Mong đợi: in ra v18.x trở lên (ví dụ v20.11.1). Nếu báo "command not found" → bạn chưa có Node.
💡 Chưa có Node? Tải bản LTS tại nodejs.org → cài như app bình thường (next-next-finish) → mở lại terminal → chạy lại lệnh trên.
b

Claude Code hoặc Claude Desktop

Chỉ cần một trong hai. Bạn sẽ chọn đường tương ứng ở Bước 4.

  • Claude Code (công cụ dòng lệnh) — hợp với ai quen terminal. code.claude.com/docs
  • Claude Desktop (app máy tính, có giao diện) — hợp với người mới, gần như không cần terminal. claude.ai/download
c

~15 phút và một chút tò mò 🙂

Không cần cài gì thêm: npx -y sẽ tự tải Filesystem server lần đầu chạy.

3Tạo thư mục "sân chơi" (sandbox)

Ta sẽ cho Claude quyền truy cập đúng một thư mục mới, trống — không phải cả ổ đĩa. Đây là thói quen an toàn quan trọng (xem Bước 8).

Windows · PowerShell 
mkdir $HOME\Documents\mcp-lab
Set-Content $HOME\Documents\mcp-lab\note1.md "# Ghi chu 1`nMua sua, trung, ca phe cho cuoi tuan."
Set-Content $HOME\Documents\mcp-lab\note2.md "# Ghi chu 2`nHop team thu Ba. Deadline bao cao thu Sau."
macOS / Linux · Terminal 
mkdir -p ~/Documents/mcp-lab
echo "# Ghi chu 1\nMua sua, trung, ca phe cho cuoi tuan." > ~/Documents/mcp-lab/note1.md
echo "# Ghi chu 2\nHop team thu Ba. Deadline bao cao thu Sau." > ~/Documents/mcp-lab/note2.md
💡 Không thích gõ lệnh? Cứ mở File Explorer / Finder, tạo thủ công thư mục mcp-lab trong Documents, rồi kéo vào đó 1–2 file .txt hoặc .md bất kỳ (ghi chú cũ cũng được). Mục tiêu chỉ là "có vài file để Claude đọc".
ℹ️ Ghi lại đường dẫn đầy đủ của thư mục — bạn sẽ dán nó ở Bước 4. Windows thường là C:\Users\<Tên-bạn>\Documents\mcp-lab; macOS là /Users/<ten-ban>/Documents/mcp-lab.

4Cắm server — chọn một trong hai đường

🅰️ Claude Code (CLI)

Cho ai quen dùng terminal · nhanh nhất
↓ Đi tới Đường A

🅱️ Claude Desktop (GUI)

Cho người mới · chỉ sửa 1 file cấu hình
↓ Đi tới Đường B

🅰️ Đường A — Claude Code (CLI)

Một câu lệnh duy nhất. Thay đường dẫn ví dụ bằng đường dẫn thư mục của bạn ở Bước 3.

Windows · PowerShell 
claude mcp add --transport stdio filesystem -- npx -y @modelcontextprotocol/server-filesystem C:\Users\Ban\Documents\mcp-lab
macOS / Linux · Terminal 
claude mcp add --transport stdio filesystem -- npx -y @modelcontextprotocol/server-filesystem ~/Documents/mcp-lab
ℹ️ Đọc cấu trúc lệnh: mọi tùy chọn (--transport, --scope…) đứng trước tên server; dấu -- ngăn cách tên với lệnh chạy server. filesystem là tên bạn tự đặt. Mặc định server chỉ áp dụng cho thư mục project hiện tại (--scope local).
💡 Muốn dùng ở mọi project? Thêm --scope user ngay sau add:
claude mcp add --scope user --transport stdio filesystem -- npx -y @modelcontextprotocol/server-filesystem C:\Users\Ban\Documents\mcp-lab

👉 Tiếp theo: Bước 5 — Kiểm tra.

🅱️ Đường B — Claude Desktop (GUI)

  1. Mở file cấu hình: trong Claude Desktop vào Settings → Developer → Edit Config. Nút này tạo/mở file claude_desktop_config.json.
    ℹ️ Vị trí file (nếu muốn mở tay):
    • Windows: %APPDATA%\Claude\claude_desktop_config.json
    • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
  2. Dán nội dung sau (thay đường dẫn cho đúng máy bạn), rồi Lưu:
Windows · claude_desktop_config.json 
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "C:\\Users\\Ban\\Documents\\mcp-lab"
      ]
    }
  }
}
macOS · claude_desktop_config.json 
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/ban/Documents/mcp-lab"
      ]
    }
  }
}
⚠️ Bẫy đường dẫn Windows: trong JSON, dấu \ phải viết đôi thành \\ (như trên). Hoặc dùng dấu gạch xuôi cho gọn: "C:/Users/Ban/Documents/mcp-lab". Luôn dùng đường dẫn tuyệt đối, không dùng .\ hay ~ rút gọn trong JSON.
  1. Thoát hẳn Claude Desktop rồi mở lại (đóng cửa sổ thôi là chưa đủ — phải quit hẳn để app đọc lại cấu hình).

👉 Tiếp theo: Bước 5 — Kiểm tra.

5Kiểm tra server đã kết nối

Nếu bạn dùng Claude Code

Trong một phiên Claude Code, gõ:

/mcp
✅ Mong đợi: thấy filesystem trong danh sách, trạng thái connected, kèm các tool (list_directory, read_text_file, write_file…). Hoặc kiểm tra ngoài phiên bằng:
claude mcp list
claude mcp get filesystem

Nếu bạn dùng Claude Desktop

Sau khi mở lại app, nhìn góc dưới-phải ô chat: sẽ có một biểu tượng MCP/công cụ (slider). Bấm vào đó để xem danh sách tool mà filesystem cung cấp.

✅ Mong đợi: thấy nhóm tool từ server filesystem. Nếu chưa thấy → xem Gỡ rối.
ℹ️ Đây chính là "list": Claude vừa làm initialize (bắt tay) rồi list (hỏi server có tool gì). Bạn đang nhìn lifecycle MCP chạy thật.

6Thử nghiệm — 3 câu lệnh "wow"

Hỏi Claude bằng tiếng Việt như bình thường. Khi cần, Claude sẽ tự gọi tool của filesystem (có thể hỏi bạn xác nhận trước khi đọc/ghi — cứ đồng ý).

1

Liệt kê file

Liệt kê các file trong thư mục mcp-lab của tôi.
✅ Claude gọi list_directory và trả về note1.md, note2.md.
2

Đọc và tóm tắt

Đọc tất cả file trong thư mục mcp-lab và tóm tắt giúp tôi.
✅ Claude gọi read_text_file / read_multiple_files rồi tóm tắt nội dung 2 ghi chú.
3

Tạo file mới (ghi)

Tạo file summary.md trong thư mục mcp-lab, tổng hợp tất cả ghi chú thành một danh sách việc cần làm.
✅ Claude gọi write_file. Mở thư mục mcp-lab → bạn thấy summary.md mới xuất hiện. 🎉 Bạn vừa cho AI "đôi tay" trên thư mục của mình — một cách an toàn, có giới hạn.

8An toàn và dọn dẹp

🔒 Vì sao chỉ trỏ vào 1 thư mục sandbox: Filesystem server có cả tool ghi/xoá/di chuyển file. Nó chỉ được phép thao tác trong những thư mục bạn liệt kê khi khởi động — không đụng được phần còn lại của máy. Vì vậy: đừng trỏ vào cả ổ C:\, thư mục Home, hay thư mục chứa secret/khóa. Cho ít quyền nhất có thể = least privilege (bài học B4/§4).

Gỡ server khi xong

Claude Code:

claude mcp remove filesystem

Claude Desktop: mở lại claude_desktop_config.json, xoá khối "filesystem": { ... } (hoặc cả mcpServers nếu chỉ có nó), lưu, rồi quit và mở lại app.

9Gỡ rối (Troubleshooting)

Triệu chứngNguyên nhân và cách xử lý
Không thấy server / tool sau khi càiClaude Desktop: bạn chưa quit hẳn rồi mở lại. Claude Code: chạy claude mcp list để xác nhận đã add đúng.
npx báo "command not found" / spawn lỗiChưa cài Node, hoặc terminal chưa nạp PATH mới. Chạy node --version; cài lại Node LTS; mở terminal/app mới.
Windows: Desktop báo lỗi spawn npx (ENOENT)Một số máy Windows cần gọi qua cmd. Thử đổi trong JSON:
"command": "cmd", "args": ["/c", "npx", "-y", "@modelcontextprotocol/server-filesystem", "C:\\...\\mcp-lab"]
Server chạy nhưng không có tool / "access denied"Sai đường dẫn, hoặc bạn không có quyền đọc/ghi thư mục đó. Dùng đường dẫn tuyệt đối, đúng thật, và là thư mục của chính bạn.
JSON không nạp đượcSai cú pháp (thừa dấu phẩy, thiếu ngoặc). Dán vào jsonlint.com để soi. Nhớ \\ cho đường dẫn Windows.
Muốn xem log để chẩn đoánDesktop · Windows: %APPDATA%\Claude\logs\mcp*.log · macOS: ~/Library/Logs/Claude/mcp*.log
Nghi server tự nó hỏngChạy thẳng để xem nó khởi động không:
npx -y @modelcontextprotocol/server-filesystem C:\Users\Ban\Documents\mcp-lab

10Thử thách thêm (tùy chọn)

  • Nhiều thư mục: thêm vài đường dẫn nữa vào cuối lệnh/args — server cho phép một danh sách thư mục: ... server-filesystem DIR1 DIR2 DIR3.
  • Hỏi tìm kiếm: "Tìm trong mcp-lab file nào có chữ deadline" → Claude dùng search_files.
  • Cây thư mục: "Vẽ cây thư mục mcp-lab" → directory_tree.
  • So sánh với §3: đọc lại repo hands-on/blockify-ip-mcp/ — đó là một MCP bạn build, khác hẳn việc chỉ cắm một server có sẵn như lab này.
🎯 Bạn vừa hoàn thành: kết nối & sử dụng MCP server đầu tiên, thấy lifecycle initialize→list→call chạy thật, và hiểu vì sao tool-surface là ranh giới bảo mật. Tiếp theo trong buổi: §4 (sự cố bảo mật MCP thật) → §5 (model và cost) → §6–§7 (observability và memory).
